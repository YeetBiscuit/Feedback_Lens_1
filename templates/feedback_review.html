Question 1: Explain the Singleton Pattern

The Singleton pattern is a creational design pattern that ensures a class has only one instance and provides a global point of access to it. This is useful when exactly one object is needed to coordinate actions across the system.

The pattern works by making the constructor private, preventing direct instantiation, and providing a static method that returns the single instance. The instance is typically stored as a static member variable.

Common use cases include configuration managers, logging systems, and connection pools where having multiple instances would be problematic or wasteful of resources.
