For instance, rather than asking a user to remember a product code from one page and type it into another, you should let them select the product from a visible list. This shifts the burden from recall, which is hard, to recognition, which is much easier. We also apply the concept of chunking to deal with Miller's Law. If you need to present a long string of numbers, like a credit card or phone number, break it into smaller groups. This allows the user to store the information as fewer chunks in their working memory, making it easier to process and verify. However, there is nuance here that we must acknowledge. Cognitive limits are not static across all users or all contexts. Expertise plays a massive role. An expert user has built up schemas in their long-term memory that allow them to chunk information more effectively than a novice. A command line interface might overwhelm a novice because every command is a new item to memorize, but an expert sees patterns and structures that reduce the cognitive load. Furthermore, context matters. If a user is under stress, or if they are multitasking, their available working memory capacity shrinks. Designing for a high-stress environment, like a hospital emergency room system, requires much stricter adherence to cognitive limits than designing for a leisure app used at home. We also have to consider that attention is driven by goals. Users often suffer from banner blindness where they ignore anything that looks like an advertisement because their goal is to find content, not promotions. Understanding this helps us place critical information where the user's goal-directed attention will actually land. Consistency allows users to rely on their long-term memory. If a button behaves differently on every page, the user cannot build a mental model of the system and must relearn the interaction every time, which spikes cognitive load. You should also design for error prevention. Since working memory is fragile, users will make mistakes.

banner that says ?No internet connection.? However, the app does not
clearly explain which features are still available offline. For example,
downloaded playlists still work, but the search function does not. This
violates the heuristic of ?Help Users Recognize, Diagnose, and Recover
from Errors?. The error message is too generic and does not help the user
understand what they can do next. The severity is rated as high because
losing connection is a frequent occurrence on mobile devices, and the
poor feedback leaves users feeling stuck and frustrated. This directly
impacts their satisfaction.
Issue 3: Small Touch Targets in Playlist Editing
When editing a playlist to remove multiple songs, the ?Edit? button and
the small red minus icons are very small. It is easy to accidentally tap
the wrong song or miss the button entirely. This violates the heuristic
of ?Error Prevention?. The design should make it harder for users to make
mistakes, especially on a mobile device where fingers are imprecise. The
severity is rated as medium because while it is annoying, the consequence
of accidentally removing the wrong song is not catastrophic; it can be
undone. However, it happens frequently and slows down the task of
playlist management.
Issue 4: Hidden Logout Option
The option to log out of the account is buried deep within the settings
menu under ?Account? and then requires scrolling to the very bottom. This
is not a frequent task, but for a new user trying to switch accounts, it
is very difficult to find. This violates the heuristic of ?Flexibility
and Efficiency of Use?. The system does not accommodate users who might
need to perform this uncommon but important action quickly. The severity
is rated as low because most users stay logged in permanently. The impact
is limited to a small subset of users, but for them, it is a significant
point of friction.
Issue 5: Autoplay Toggle is Confusing
After a playlist or album finishes, Spotify automatically plays similar
songs. There is a toggle to turn this off, but its location is not
obvious. It is hidden in the playback settings, and the label ?Autoplay?
does not clearly explain that it refers to playing similar music after
your selection ends. This violates the heuristic of ?Match Between System
and the Real World?. The term ?Autoplay? is system-oriented jargon. A
more user-friendly term would be clearer. The severity is rated as low
because the feature itself is not critical, but the lack of clarity means

users might not realize they can control this behaviour, leading to
unexpected data usage or unwanted music.
Recommendations
Based on the findings, the following design changes are recommended to
improve the usability of the Spotify app.
For Issue 1, the queue button should be placed in a consistent location
on the ?Now Playing? screen, perhaps always in the bottom-right corner.
This would fix the consistency problem and make it easier for users to
build a mental model of the interface. This relates to the course
material on human cognition, where consistency helps users rely on
long-term memory instead of having to relearn the interface each time.
For Issue 2, the offline error message should be more specific. Instead
of just saying ?No internet connection,? it should say something like
?You are offline. You can still play downloaded music. Search and radio
are unavailable.? This helps users diagnose the problem and recover by
knowing their options. This directly applies the heuristic of helping
users with errors.
For Issue 3, the touch targets for the edit and delete buttons in
playlists should be made larger. The minimum recommended size for touch
targets is about 44x44 pixels. Making them bigger would prevent
accidental taps and make the editing process smoother. This is a simple
fix that prevents errors before they happen.
For Issue 4, a quick-action button for logging out could be added to the
main settings page, or the account section could be reorganized so the
logout button is more prominent. This would improve efficiency for the
small number of users who need to switch accounts.
For Issue 5, the ?Autoplay? toggle should be renamed to ?Play similar
music when your music ends.? This uses plain language that matches the
user?s real-world understanding. The toggle could also be placed in a
more visible area, such as at the top of the ?Now Playing? screen when a
queue is about to end.

Between System and the Real World?. The recommendations provided are
